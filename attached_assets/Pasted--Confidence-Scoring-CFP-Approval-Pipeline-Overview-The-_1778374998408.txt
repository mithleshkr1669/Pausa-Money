# Confidence Scoring & CFP Approval Pipeline

## Overview

The confidence scoring system ensures AI plans reach the right human reviewer — routing ~60% to auto-approve, ~35% to human review, and ~5% to paid consultation. This keeps CFP workload manageable while maintaining quality.

## Scoring Dimensions (weighted)

| Dimension | Weight | Low score means... | High score means... |
|-----------|--------|--------------------|---------------------|
| situation_completeness | 1.5× | User gave vague info | Full income/debt/goals provided |
| topic_risk | 1.4× | Pure budgeting/savings | Securities/stock advice |
| complexity | 1.3× | 26yo, first job, basic SIP | NRI, HUF, ESOP, inheritance |
| plan_consistency | 1.2× | Logic contradictions found | Plan internally sound |
| regulatory_sensitivity | 1.4× | No regulated territory | Directly recommends securities |
| historical_similarity | 1.1× | Unusual situation | Matches 1000+ approved plans |

**Weighted average formula:**
```typescript
const weights = { situation_completeness: 1.5, topic_risk: 1.4, complexity: 1.3, plan_consistency: 1.2, regulatory_sensitivity: 1.4, historical_similarity: 1.1 }
const totalWeight = Object.values(weights).reduce((a, b) => a + b, 0)
const weightedAvg = Object.entries(scores).reduce((sum, [key, val]) => sum + val * weights[key], 0) / totalWeight
```

## Routing Logic

```typescript
// lib/utils/routing.ts
export type RoutingDecision = {
  tier: 1 | 2 | 3 | 4 | 5
  routing: 'auto_approve' | 'community_queue' | 'cfp_queue' | 'senior_cfp' | 'paid_consultation'
  sla: string
  userMessage: string
}

export function routePlan(score: number, flags: string[]): RoutingDecision {
  // Hard flags always escalate regardless of score
  const hardFlagPatterns = ['NRI', 'DTAA', 'HUF', 'ESOP', 'F&O', 'derivatives', 'chit fund', 'inheritance']
  const hasHardFlag = flags.some(f => hardFlagPatterns.some(p => f.toLowerCase().includes(p.toLowerCase())))

  if (hasHardFlag) {
    return {
      tier: 5,
      routing: 'paid_consultation',
      sla: 'Schedule a session',
      userMessage: 'Your situation involves specialised tax and legal areas. We recommend a 1:1 session with a SEBI-registered expert.'
    }
  }

  if (score >= 85) return {
    tier: 1, routing: 'auto_approve', sla: 'Instant',
    userMessage: 'Your plan is ready!'
  }
  if (score >= 70) return {
    tier: 2, routing: 'community_queue', sla: '~4 hours',
    userMessage: 'Your plan is being reviewed by our community experts. Ready in ~4 hours.'
  }
  if (score >= 55) return {
    tier: 3, routing: 'cfp_queue', sla: '~2 hours',
    userMessage: 'Your plan is being reviewed by a certified CFP. Ready in ~2 hours.'
  }
  if (score >= 40) return {
    tier: 4, routing: 'senior_cfp', sla: '~30 minutes',
    userMessage: 'Complex situation detected. A senior SEBI-registered adviser is reviewing this. Ready in ~30 minutes.'
  }
  return {
    tier: 5, routing: 'paid_consultation', sla: 'Schedule session',
    userMessage: 'Your financial situation needs personalised expert attention. Please book a 1:1 session.'
  }
}
```

## CFP Review Dashboard UX

### 90-Second Review Design
The dashboard is engineered for a CFP to review a standard plan in under 90 seconds:

1. **Pre-structured summary card** (not raw text) — shows: Income ₹Xk | Debt ₹Xk | Goals: [list] — CFP absorbs in 10 seconds
2. **Flagged items highlighted automatically** — red border on steps that triggered flags
3. **One-click Approve** OR **inline edit** — click step text directly, edit, save. No modal, no form.
4. **Edit reason tags** (not free text) — [Wrong instrument] [Tax error] [Better alternative] [Risk too high] [Outdated rate]
5. **Batch mode** — identical-profile plans grouped. Approve all with one click.

### CFP Dashboard Route
```
app/
└── (cfp)/
    ├── review/page.tsx          # Queue of plans awaiting review
    ├── review/[id]/page.tsx     # Single plan review with inline edit
    └── review/batch/page.tsx    # Batch review for similar plans
```

## Audit Sampling

5% of auto-approved plans are randomly sampled weekly for CFP audit review. This:
- Maintains quality baseline
- Catches AI errors that scored high confidence incorrectly
- Generates training data for confidence model improvement

```typescript
// After auto_approve routing

  await supabase.from('audit_queue').insert({
    plan_id: planId,
    audit_reason: 'random_sample',
    due_by: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000) // 7 days

}
```
