# FinCoach India — Platform Build Skill

## What You Are Building

FinCoach is an **India-first AI financial co-pilot + community platform**. Two products in one:

1. **AI Co-pilot** — User describes their situation in Hindi or English → gets a personalised, numbered action plan in seconds. India-specific: PPF, NPS, ELSS, Section 80C, EPF, ITR, CIBIL.
2. **Community Hub** — Structured Q&A with upvotes, verified expert answers, AI-drafted first responses, and a 5-tier inner ring membership system.

**Target user:** Indians aged 22–38, salaried or freelance, Tier 1–3 cities. Currently asking financial questions on r/personalfinanceindia and getting contradictory opinions. They want decisions, not more content.

**Core moat:** India-specificity (no US competitor understands PPF/NPS/ELSS), Hindi-first, and CFP-verified answers.

## Database Schema

Always use this schema. Do not invent new tables without checking here first.

```sql
-- USERS (extends Supabase auth.users)
CREATE TABLE profiles (
  id            uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  username      text UNIQUE,
  display_name  text,
  avatar_url    text,

  city_tier     text,                    -- metro | tier2 | tier3
  language_pref text DEFAULT 'en',       -- en | hi
  income_range  text,                    -- stored as range, never exact number
  created_at    timestamptz DEFAULT now(),
  updated_at    timestamptz DEFAULT now()
);

-- PLANS (AI-generated financial plans)
CREATE TABLE plans (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid REFERENCES profiles(id),
  situation_text  text NOT NULL,
  ai_plan         jsonb NOT NULL,         -- structured plan JSON
  confidence      int NOT NULL,           -- 0-100
  tier            int NOT NULL,           -- 1=auto 2=community 3=cfp 4=senior 5=consult
  flags           text[],                 -- flagged issues from confidence scorer
  status          text DEFAULT 'pending', -- pending|approved|modified|escalated|consultation_required
  routing         text NOT NULL,          -- auto_approve|community_queue|cfp_queue|senior_cfp|paid_consultation
  is_public       bool DEFAULT false,
  share_slug      text UNIQUE,            -- for public share links
  language        text DEFAULT 'en',
  created_at      timestamptz DEFAULT now()
);

-- PLAN REVIEWS (CFP + community validator reviews)
CREATE TABLE plan_reviews (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  plan_id         uuid REFERENCES plans(id),
  reviewer_id     uuid REFERENCES profiles(id),
  reviewer_type   text,     -- community_validator | cfp | senior_cfp
  action          text,     -- approved | modified | escalated
  original_plan   jsonb,
  modified_plan   jsonb,    -- null if approved without changes
  edit_tags       text[],   -- [wrong_instrument, tax_error, better_alternative, outdated_rate]
  review_time_s   int,      -- seconds taken
  pay_amount_paise int,     -- in paise (1 rupee = 100 paise)
  created_at      timestamptz DEFAULT now()

-- POSTS (community Q&A)
CREATE TABLE posts (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       uuid REFERENCES profiles(id),

  body          text NOT NULL,           -- rich text (Tiptap JSON stored as text)

  upvotes       int DEFAULT 0,
  view_count    int DEFAULT 0,
  is_answered   bool DEFAULT false,
  language      text DEFAULT 'en',       -- en | hi
  slug          text UNIQUE,
  created_at    timestamptz DEFAULT now()
);

-- ANSWERS
CREATE TABLE answers (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id       uuid REFERENCES posts(id),
  user_id       uuid REFERENCES profiles(id),
  body          text NOT NULL,
  upvotes       int DEFAULT 0,
  is_ai         bool DEFAULT false,      -- AI-drafted first answer
  is_verified   bool DEFAULT false,      -- verified by CFP/SEBI RIA

);

-- COMMENTS
CREATE TABLE comments (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  answer_id   uuid REFERENCES answers(id),
  user_id     uuid REFERENCES profiles(id),

  created_at  timestamptz DEFAULT now()

-- VOTES (upvote tracking — one per user per item)
CREATE TABLE votes (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid REFERENCES profiles(id),
  target_id   uuid NOT NULL,
  target_type text NOT NULL,   -- post | answer

-- EXPERTS (verified CFPs, SEBI RIAs, CAs)
CREATE TABLE experts (
  id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),

  session_price_paise int,           -- in paise

-- TRAINING DATA (for future fine-tuning)
CREATE TABLE training_data (
  id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),

-- FINCOIN LEDGER
CREATE TABLE fincoin_ledger (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid REFERENCES profiles(id),

  reason      text NOT NULL,          -- helpful_answer | plan_review | referral | pro_redemption

  created_at  timestamptz DEFAULT now()

ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
ALTER TABLE answers ENABLE ROW LEVEL SECURITY;
ALTER TABLE comments ENABLE ROW LEVEL SECURITY;

-- Key RLS policies (add more as needed)
```

```typescript
// lib/prompts/india-finance.ts
export const INDIA_FINANCE_SYSTEM_PROMPT = `You are FinCoach, an expert AI financial co-pilot built specifically for India. You have deep knowledge of:

INDIAN TAX SYSTEM:
- Section 80C (₹1.5L limit): ELSS, PPF, EPF, NSC, tax-saving FD, life insurance premium, home loan principal
- Section 80D: Health insurance (₹25k self+family, ₹50k for senior citizen parents)
- Section 80CCD(1B): Additional ₹50k NPS deduction over 80C limit
- Old vs new tax regime: Break-even analysis, which is better at each income bracket
- ITR forms: ITR-1 (salaried), ITR-2 (capital gains), ITR-3 (business/freelance)
- Advance tax: Who pays, when (15 June, 15 Sept, 15 Dec, 15 Mar), how much
- LTCG: Equity >₹1L gains taxed at 10%, Debt at income slab rate (post 2023 budget)

INDIAN INVESTMENT INSTRUMENTS:
- PPF: 15-year lock-in, ₹1.5L/year max, currently 7.1% p.a., tax-free maturity, partial withdrawal rules
- NPS: Tier 1 (pension, 60% annuity at retirement) vs Tier 2 (flexible withdrawal), equity allocation by age, eNPS registration
- ELSS: 3-year lock-in, equity mutual fund with 80C benefit, recommend only direct plans
- EPF: Employee 12% + Employer 12%, VPF option, EPFO withdrawal rules, UAN
- Sukanya Samriddhi Yojana: For girl child, 8.2% current rate, 21-year maturity
- Senior Citizens Savings Scheme (SCSS): 8.2%, quarterly payout, max ₹30L
- RBI Floating Rate Bonds: Currently 8.05%, taxable
- Sovereign Gold Bonds (SGB): 2.5% p.a. + gold price appreciation, 8-year tenure, tax-free at maturity if held to maturity
- Gold ETF vs SGB vs physical gold: Trade-offs clearly explained
- Direct vs Regular mutual funds: Direct always better (no commission), show expense ratio difference

DEBT & LOANS:
- Home loan: LTV ratios (80-90%), CIBIL score requirements (750+), home loan interest deduction under Section 24 (₹2L limit)
- Personal loan: Typically 12-24% interest, avalanche vs snowball method
- Credit card: 36-42% annual interest, always pay full amount
- Gold loan: Lower interest (10-15%) secured, useful alternative
- CIBIL score: Factors, recovery timeline, hard vs soft inquiry

INSURANCE (INDIA-SPECIFIC):
- Term life: Only pure term plans (not ULIPs, not endowment, not money-back)
- Coverage: 10-15x annual income minimum
- Best pure term insurers: HDFC Click2Protect, ICICI iProtect, Tata AIA — compare on claim settlement ratio
- Health insurance: Individual vs family floater, restore benefit, waiting periods
- NEVER recommend LIC endowment/money-back/ULIP as investment

FREELANCER/SELF-EMPLOYED SPECIFICS:
- GST: Mandatory registration above ₹20L turnover (₹10L for some states and service providers)
- Presumptive taxation: Section 44ADA (50% of gross receipts for professionals, up to ₹75L)
- Business expenses deductible: Internet, phone, laptop, workspace rent, professional development
- No EPF as freelancer → NPS + mutual fund SIP for retirement
- Variable income budgeting: Base budget on lowest 3-month average

OUTPUT FORMAT:
Return ONLY valid JSON. No markdown, no preamble, no explanation outside JSON.
{
  "health_score": <integer 0-100>,
  "health_label": "<Healthy|Caution|Critical>",
  "health_summary": "<2 sentences specific to their numbers>",
  "actions": [
    {
      "priority": "<critical|high|medium|low>",
      "title": "<specific action title>",
      "timeline": "<Do this: today|this week|this month|in 3 months|in 6 months|in 1 year>",
      "impact": "<High impact|Medium impact|Foundation>",
      "description": "<2-3 sentences. MUST reference their actual rupee amounts. Explain why THIS matters for THEIR specific situation.>",
      "steps": [
        "<concrete actionable step with specific numbers/platforms/actions>",
        "<step 2>",
        "<step 3>"
      ]
    }
  ]
}

CRITICAL RULES:
1. NEVER provide specific stock picks or specific fund names as the only option — say "Nifty 50 Index Fund (e.g., UTI Nifty 50, HDFC Index Fund Nifty 50)"
2. Always add: this is educational guidance, not SEBI-regulated investment advice
3. Use Indian number formatting: ₹1,00,000 not ₹100,000
4. Reference their ACTUAL numbers — if income is ₹60,000 say "₹60,000 monthly income" not "your income"
5. Order actions by urgency: emergency fund → insurance → high-interest debt → tax optimisation → investing
6. For debt: always recommend clearing highest-interest first (avalanche method) unless psychological argument for snowball
7. Return 5-7 actions. Not more. Quality over quantity.
8. If user writes in Hindi, respond with Hindi text inside JSON strings.`;
```

## Legal Constraints (Must Follow in All Code)

1. **SEBI disclaimer** — add to every AI plan output, every page that shows financial guidance:
   ```typescript
   export const SEBI_DISCLAIMER = "This content is generated by AI for educational purposes only and does not constitute investment advice under SEBI (Investment Advisers) Regulations, 2013. For personalised securities advice, consult a SEBI-registered Investment Adviser."
   ```

2. **Never use the word "advice"** in UI copy. Use: guidance, suggestions, plan, recommendations.

3. **Expert credential verification** — before displaying any expert's badge, check `experts.verified = true`. Never show unverified experts as verified.

4. **AI disclosure** — per SEBI January 2025 rule, all AI-generated content must be labelled "Generated by AI".

5. **Data collection consent** — before storing any financial data (income, debt, goals), show consent checkbox. Store `consented: true` in training_data table.

6. **Delete my data** — every user profile page must have a "Delete account and all data" button that triggers a cascade delete.

7. **Rate limiting** — all AI API routes must be rate-limited: 5 requests/minute per user (free tier), 20/minute (pro tier). Use Upstash Redis.

## API Route Patterns

### AI Plan Generation (Streaming)

```typescript
// app/api/plan/route.ts
import Anthropic from '@anthropic-ai/sdk'
import { INDIA_FINANCE_SYSTEM_PROMPT } from '@/lib/prompts/india-finance'

export async function POST(req: Request) {
  const { income, expenses, savings, debt, age, goals, situation, employment, language } = await req.json()

  // Rate limit check (Upstash Redis)
  // ... rate limit logic ...

  const userMessage = buildUserMessage({ income, expenses, savings, debt, age, goals, situation, employment })

  const client = new Anthropic()  // uses ANTHROPIC_API_KEY env var

  const stream = await client.messages.stream({
    model: 'claude-haiku-4-5-20251001',  // use Sonnet for Pro users
    max_tokens: 2000,
    system: INDIA_FINANCE_SYSTEM_PROMPT,
    messages: [{ role: 'user', content: userMessage }]
  })

  return new Response(stream.toReadableStream(), {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
    }
  })
}

function buildUserMessage(data: any): string {
  const savingsRate = data.income && data.expenses
    ? Math.round(((data.income - data.expenses) / data.income) * 100)
    : null

  return `FINANCIAL PROFILE:
- Age: ${data.age || 'not provided'} years
- Employment: ${data.employment}
- Monthly income: ₹${Number(data.income).toLocaleString('en-IN')}
- Monthly expenses: ₹${Number(data.expenses).toLocaleString('en-IN')}
- Savings potential: ₹${Number(data.income - data.expenses).toLocaleString('en-IN')}/month (${savingsRate}% savings rate)
- Total savings/investments: ₹${Number(data.savings).toLocaleString('en-IN')}
- Total debt: ₹${Number(data.debt).toLocaleString('en-IN')}
- Goals: ${data.goals?.join(', ') || 'not specified'}
${data.situation ? `\nAdditional context: "${data.situation}"` : ''}

Generate a personalised financial action plan. Reference actual rupee amounts. Use India-specific instruments.`
}
```

## Common Mistakes to Avoid

1. **Never expose `ANTHROPIC_API_KEY` to the browser.** All AI calls go through `/api/` routes.
2. **Never store exact income/salary numbers** — store ranges (₹40k–₹60k) or use them transiently.
3. **Never skip the SEBI disclaimer** on any page showing financial guidance.
4. **Never call CFP-verified without checking `experts.verified = true`** in DB.
5. **Never use `regular` mutual fund links** in affiliate recommendations — always `direct` plans.
6. **Never hardcode rupee amounts** from the system prompt into UI — always pull from AI response.
7. **Always use Indian number formatting** — `toLocaleString('en-IN')` — not Western (1,000,000 → 10,00,000).
8. **Always rate-limit AI routes** — unprotected AI routes will be abused and cost money fast.
